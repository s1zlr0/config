Эмулятор командной строки VFS
1. Общее описание
Эмулятор командной строки VFS (Virtual File System) - это графическое приложение, имитирующее работу командной оболочки UNIX-подобных операционных систем. Проект реализован на Python с использованием библиотеки tkinter для графического интерфейса.

Основные возможности:

Интерактивный режим работы (REPL)

Поддержка параметров командной строки

Выполнение стартовых скриптов

Обработка ошибок и валидация ввода

Кросс-платформенная работа (Windows, Linux, Mac)

2. Описание всех функций и настроек
Класс ShellEmulator
Основной класс эмулятора, реализующий весь функционал.

Методы:
init(root, vfs_path=None, startup_script=None)

Инициализация GUI и параметров эмулятора

root: главное окно tkinter

vfs_path: путь к виртуальной файловой системе

startup_script: путь к стартовому скрипту

execute_command(event)

Обработчик ввода команд из интерфейса

Выполняется при нажатии Enter в поле ввода

execute_script(script_path)

Выполнение скрипта с поддержкой комментариев

Имитирует диалог с пользователем

parse_command(command_string)

Парсер команд, разделяющий ввод на команду и аргументы

Возвращает кортеж (command, args)

print_output(text)

Вывод текста в область консоли

Автоматическая прокрутка к новому контенту

Обработчики команд
cmd_exit(args)

Завершение работы эмулятора

Использование: exit

cmd_ls(args) - ЗАГЛУШКА

Имитация вывода содержимого директории

Использование: ls или ls [аргументы]

Всегда выводит сообщение о заглушке

cmd_cd(args) - ЗАГЛУШКА

Имитация смены директории

Использование: cd [путь] или cd

Валидация количества аргументов

Параметры командной строки
--vfs-path <путь>

Указывает физический путь к директории VFS

По умолчанию: ./vfs_root

--startup-script <путь>

Задает путь к скрипту для автоматического выполнения при запуске
3. Описание команд для сборки проекта и запуска тестов
ls
